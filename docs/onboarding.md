# TrustNAS — User Onboarding Guide

## Overview

TrustNAS is a self-hosted NAS built on [RustFS](https://github.com/rustfs/rustfs) (S3-compatible storage) with [Tailscale](https://tailscale.com) for secure networking. Each user gets a private S3 bucket with a hard quota, accessible only by their own IAM credentials.

The `scripts/create_bucket.py` script automates the admin side of onboarding:

- Creates the S3 bucket
- Sets a storage quota
- Applies a bucket policy that restricts access to a single IAM user
- Generates a random password for that user

After running the script, the admin creates the IAM user in the RustFS web console and hands the credentials to the user.

## Prerequisites

- Docker services running (`docker compose up -d`)
- RustFS console reachable at `http://<host>:9001`
- Python 3.13+ with dependencies installed (see [uv](https://docs.astral.sh/uv/) or `pip`):
  ```bash
  pip install boto3 requests
  ```
- Environment file configured:
  ```bash
  cp .env.example .env
  # edit .env with your actual credentials
  ```

## Creating a Bucket

Use [`scripts/create_bucket.py`](../scripts/create_bucket.py) with three required arguments:

| Flag           | Description                        | Example                |
|----------------|------------------------------------|------------------------|
| `--bucket`     | Name of the S3 bucket              | `alice-files`          |
| `--username`   | IAM username that will own it      | `alice`                |
| `--quota`      | Storage limit with optional units  | `10GiB`, `500MB`, `1TB`|

Supported quota units: `B`, `KB`, `MB`, `GB`, `TB`, `KiB`, `MiB`, `GiB`, `TiB`.

Optional flags for non-default setups:

| Flag                  | Default                | Description                       |
|-----------------------|------------------------|-----------------------------------|
| `--endpoint`          | `http://rustfs-server:9000` | RustFS S3/admin endpoint     |
| `--region`            | `us-east-1`            | AWS region string                 |
| `--admin-access-key`  | `$RUSTFS_ACCESS_KEY` (from `.env`) | Admin access key             |
| `--admin-secret-key`  | `$RUSTFS_SECRET_KEY` (from `.env`) | Admin secret key             |

### Example

```bash
python scripts/create_bucket.py \
  --bucket alice-docs \
  --username alice \
  --quota 50GiB
```

The script outputs a summary table, including a generated password. Copy the password — it is only shown once.

## Creating the IAM User

The bucket policy is already in place after the script runs, but the IAM user itself does not yet exist. Create it:

1. Open the RustFS console at `http://<host>:9001`
2. Navigate to **Users → Add User**
3. Enter the **username** and **password** printed by the script
4. Save the user

The user can now connect to their bucket using any S3-compatible client with these details:

- **Endpoint:** the RustFS S3 endpoint (e.g. `http://<host>:9000` or the Tailscale host/port)
- **Access Key:** the IAM username
- **Secret Key:** the password generated by the script
- **Region:** `us-east-1` (or whatever was passed to `--region`)
